# GitHub Copilot Instructions: Macify Design

Use Macify Design for web tool/workspace product UI in this repository when the requested workflow is operational: create, edit, manage, publish, configure, analyze, monitor, review, automate, debug, generate, preview, or control objects.

Prefer a single Mac-like desktop workbench with semantic light/dark tokens, app window, toolbar/titlebar, sidebar, main workspace, optional inspector, sheets/popovers, compact controls, and status feedback.

Respect the existing project architecture, framework, components, icons, and CSS/token system. Add Mac-like skin as a layer; do not rewrite unrelated architecture or add heavy dependencies unless requested.

Mobile must be a responsive app shell, not a scaled desktop window. Collapse sidebars to drawers/navigation, inspectors to sheets/detail views, dense tables to lists/drill-ins, and keep primary actions reachable.

Implemented UI must include real states and interactions where relevant: mock data, search/filter/sort, selection, create/edit/delete, preview/publish/save/run/status feedback, empty/loading/error/success/disabled/focus states, and accessible controls.

Do not simulate macOS, add Dock or system menu bar, copy Apple apps, use Apple logos/screenshots/assets, or rely on traffic-light window dots as product controls.
