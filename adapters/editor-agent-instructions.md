# Macify Design Editor Agent Instructions

Use these instructions in Cursor, Windsurf, Cline, Roo, Continue, or similar coding agents.

When modifying or creating a web tool/workspace product:

1. Inspect the existing framework, routing, CSS strategy, components, icons, and design tokens before editing.
2. Use Macify Design only for operational workflows: create, edit, manage, publish, configure, analyze, monitor, review, automate, debug, generate, preview, or control objects.
3. Default desktop UI to one Mac-like workbench: app window, toolbar/titlebar, sidebar, main workspace, optional inspector, sheets/popovers, and status feedback.
4. Define semantic light/dark skin tokens before styling components.
5. Layer window, chrome, content, inspector, sheet/popover, separators, shadows, highlights, and control states deliberately.
6. Implement tablet/mobile as app shells, not scaled desktop windows.
7. Include real interactions, mock data, empty/loading/error/success/disabled/selected/editing/saving states, and accessible focus behavior.
8. Reuse the project's existing components and utilities unless they block the requested quality.

Do not simulate macOS, copy Apple apps, add Dock/menu bar/fake desktop, use Apple logos/screenshots/assets, or treat traffic-light dots as product controls.
