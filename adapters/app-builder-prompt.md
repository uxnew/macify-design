# Macify Design App Builder Prompt

Use this prompt only when the requested product is a web tool/workspace workflow: users create, edit, manage, publish, configure, analyze, monitor, review, automate, debug, generate, preview, or control objects.

If the request is primarily a marketing page, passive reading site, blog, portfolio, ecommerce storefront, game, or content showcase, do not apply Macify Design unless the user explicitly asks for a Mac-like tool interface.

Build qualifying requests as usable web tool/workspace products with Macify Design.

Do not make a marketing page or passive mockup. Create an interactive product surface with realistic sample data, primary workflows, real states, and responsive behavior.

Desktop:

- Use a Mac-like windowed workbench.
- Include toolbar/titlebar, sidebar or source list, main workspace, optional inspector/properties panel, sheets/popovers, and status feedback when the workflow needs them.
- Define light and dark appearances with semantic skin tokens.
- Use layered materials for window, chrome, content, inspector, and overlays.

Mobile:

- Use a mobile app shell, not a scaled desktop window.
- Convert sidebar to drawer/navigation, inspector to bottom sheet/detail view, toolbar to compact top bar, and dense tables to lists or drill-ins.
- Keep primary actions, search/navigation, status, and lightweight editing usable.

Interactions:

- Include mock data.
- Implement search/filter/sort, selection, create/edit/delete, preview/publish/save/run/status feedback, or the product-specific equivalents.
- Include empty, loading, error, success, disabled, selected, editing, saving, and focus states where relevant.

Boundaries:

- Do not simulate macOS.
- Do not use Apple logo, Apple screenshots, Dock, system menu bar, fake desktop, copied Finder/Settings/Mail UI, or Apple-owned assets.
- Use Mac-like skin, material layering, compact controls, and workbench layout only.
