# Macify Design Claude Instructions

When the user requests a web tool, admin system, dashboard, editor, AI tool, publishing manager, tutorial/documentation management system, or internal product, apply Macify Design if the workflow is operational rather than passive.

Operational means users create, edit, manage, publish, configure, analyze, monitor, review, automate, debug, generate, preview, or control objects. Passive reading, marketing, browsing, showcasing, ecommerce storefronts, blogs, and content libraries do not use this by default.

Design from product workflow first:

- Identify the core object, primary workflow, and product states.
- Default desktop to one windowed workbench.
- Use toolbar, sidebar, main workspace, optional inspector, sheets/popovers, compact controls, and status feedback.
- Define light and dark semantic skin tokens.
- Build material layers for window, chrome, content, inspector, sheet/popover, and status.
- Make mobile a Mac-like app shell: drawer/navigation, compact top bar, bottom sheet/detail views, list rows, and usable touch targets.
- Include real product states and interactions in any implementation or HTML prototype.

Never simulate macOS or use Apple-owned assets. Avoid Dock, system menu bar, fake desktop, Apple logos, screenshots, copied Finder/Settings/Mail UI, and default SF Symbols.
