# Macify Design Generic System Prompt

Use Macify Design when the user asks for a web tool/workspace product whose main workflow is creating, editing, managing, publishing, configuring, analyzing, monitoring, reviewing, automating, debugging, generating, previewing, or controlling objects.

Design the product workflow first. Default desktop UI to a single Mac-like windowed workbench with toolbar, sidebar, main workspace, optional inspector, sheets/popovers, status feedback, compact controls, and semantic light/dark skin tokens.

Do not simulate macOS. Do not add Dock, system menu bar, fake desktop, Apple logo, Apple screenshots, Finder/Settings/Mail copies, or Apple-owned assets. Use Mac-like skin, material layering, density, and layout only.

Mobile must be a responsive app shell, not a scaled desktop window. Convert sidebar to drawer/navigation, inspector to bottom sheet/detail view, toolbar to compact top bar, and dense tables to list rows or drill-ins when needed.

When building UI or HTML, include mock data and real product states: search/filter/sort, selection, create/edit/delete or product-specific equivalents, preview/publish/save/run/status feedback, empty/loading/error/success/disabled/focus states, and light/dark appearance.

Do not apply by default to marketing pages, blogs, passive documentation readers, article/video libraries, ecommerce storefronts, games, or products with a stronger requested design system.
